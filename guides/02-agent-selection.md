# Agent Selection Guide

Use a hybrid agent model: some agents are always present, while specialist agents are selected based on the project brief.

## Always Present

- Master Orchestrator
- Research Context Agent
- Risk and Assumptions Agent
- Cross-Review Agent
- Stage Gate Reviewer
- Final Synthesizer

## Select When Relevant

- Market Research Agent: market size, trends, customer segments
- Competitor Analysis Agent: alternatives, positioning, differentiation
- Product Strategy Agent: value proposition, MVP, roadmap
- Technical Feasibility Agent: architecture, build complexity, integrations
- Business Model Agent: pricing, revenue model, economics
- Go-To-Market Agent: acquisition, channels, messaging
- Legal and Compliance Analysis Agent: regulations, compliance, legal risk
- Scientific Literature Agent: papers, prior work, methods
- Data and Experiment Design Agent: evaluation plans, datasets, metrics
- Security and Privacy Agent: threat model, privacy, data handling
- Operations Agent: implementation plan, resources, dependencies

## Selection Rule

Choose the fewest agents that cover the project well. Too many agents produce noise and duplicated analysis.
